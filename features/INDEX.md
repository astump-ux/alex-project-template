# Feature Index

> Central tracking for all features. Updated by skills automatically.
> **Where specs live:** each feature's spec is committed to this repo at `docs/spec/<PROJ-ID>_<slug>.md` and linked from the `Spec` column below. The repo is canonical — Google Drive holds only exports/copies. See `CLAUDE.md` §10.

## Status Legend
- **Planned** - `/requirements` done, spec written, architecture not yet designed
- **Architected** - `/architecture` done, tech design approved, ready to build
- **In Progress** - `/frontend` or `/backend` active or completed, not yet in QA
- **In Review** - `/qa` active, testing in progress
- **Approved** - `/qa` passed, no critical/high bugs, ready to deploy
- **Deployed** - `/deploy` done, live in production
